# Legal Summarization Assistant for Pakistani Law

A specialized tool for generating abstractive summaries of Pakistani legal documents, focusing on case outcomes, key facts, evidence, legal issues, and applicable statutes.

## Features

- **Abstractive Summarization**: Produces original summaries, not copy-paste extracts
- **Specialized for Pakistani Law**: Tailored for Pakistani legal context
- **Comprehensive Coverage**: Focuses on:
  - Case outcomes/results (for judgments)
  - Key facts
  - Important evidence or legal issues
  - Applicable statutes/citations
  - Quick understanding for readers
- **Neutral and Professional**: Clear, concise, and avoids legal advice

## Installation

1. Ensure you have Python 3.8 or higher installed
2. Activate your virtual environment (if using one):
   ```bash
   # On Windows
   venv\Scripts\activate
   
   # On Linux/Mac
   source venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   # From adal-backend/sum
   pip install -r ../requirements.txt
   # or lightweight WSL install:
   pip install -r ../requirements-wsl.txt
   ```

## Setup

You need an OpenAPI key to use this tool. The easiest way is to use a `.env` file:

1. **Create a `.env` file** (recommended):
   - Copy the `.env.example` file to `.env`:
     ```bash
     # On Windows (PowerShell)
     Copy-Item .env.example .env
     
     # On Windows (CMD)
     copy .env.example .env
     
     # On Linux/Mac
     cp .env.example .env
     ```
   - Open the `.env` file and replace `your-api-key-here` with your actual API key:
     ```
     OPENAPI_KEY=sk-your-actual-api-key-here
     ```
   - The `.env` file is automatically loaded when you run the script (it's already in `.gitignore` for security)

2. **Alternative: Set environment variable**:
   ```bash
   # On Windows (PowerShell)
   $env:OPENAPI_KEY="your-api-key-here"
   
   # On Windows (CMD)
   set OPENAPI_KEY=your-api-key-here
   
   # On Linux/Mac
   export OPENAPI_KEY="your-api-key-here"
   ```

3. **Or pass it via command-line** (see Usage below)

## Usage

### Command Line

Basic usage:
```bash
python legal_summarizer.py path/to/legal_document.txt
```

With custom output file:
```bash
python legal_summarizer.py input.txt -o summary.txt
```

With API key specified:
```bash
python legal_summarizer.py input.txt -k your-api-key-here
```

With custom model:
```bash
python legal_summarizer.py input.txt -m gpt-3.5-turbo
```

With custom token limit:
```bash
python legal_summarizer.py input.txt -t 3000
```

### Python API

```python
from legal_summarizer import LegalSummarizer

# Initialize with API key
summarizer = LegalSummarizer(api_key="your-api-key-here")

# Option 1: Summarize from text string
legal_text = "Your legal document text here..."
summary = summarizer.summarize(legal_text)
print(summary)

# Option 2: Summarize from file
summary = summarizer.summarize_from_file("legal_document.txt")

# Save summary to file
summarizer.save_summary(summary, "output_summary.txt")
```

## Input Format

The tool expects plain text files containing the extracted text from legal PDFs. If you have a PDF, you'll need to extract the text first using a PDF extraction tool like:

- `pdftotext` (from poppler-utils)
- Python libraries like `PyPDF2` or `pdfplumber`
- Online PDF to text converters

## Output

The summary will be:
- Saved to a file (default: `input_filename_summary.txt`)
- Printed to the console
- Structured to highlight:
  - Case outcome/decision
  - Key facts
  - Evidence and legal issues
  - Applicable statutes and citations
  - Quick understanding points

## Models

By default, the tool uses `gpt-4` for high-quality summaries. You can also use:
- `gpt-3.5-turbo` (faster, more cost-effective)
- `gpt-4-turbo-preview` (latest GPT-4 variant)

## Limitations

- Requires an OpenAPI key and internet connection
- API usage incurs costs based on the API provider's pricing
- The tool does not provide legal advice
- Summaries are generated by AI and should be reviewed for accuracy

## Example

```bash
python legal_summarizer.py judgment_2024.txt -o judgment_summary.txt
```

This will:
1. Read the legal document from `judgment_2024.txt`
2. Generate an abstractive summary
3. Save it to `judgment_summary.txt`
4. Display the summary in the console
