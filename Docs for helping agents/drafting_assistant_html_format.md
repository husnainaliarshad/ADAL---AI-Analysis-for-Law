# Drafting Assistant - HTML Format Guidelines

This document outlines the required formatting for the Legal Document Drafting Assistant output (specifically the `html_to_insert` payload generated by `drafting_service.py` via `DraftingEditor.jsx` using Tiptap / ProseMirror).

## Overview

Unlike the main chat service which operates strictly on Markdown, the **Drafting Assistant uses HTML directly** for all document content. AI endpoints dealing with the drafting interface must output their finalized text payloads in purely structured HTML. 

## Supported Tags

The Tiptap editor parses semantic HTML. Use standard tags for generating legal drafts:

1. **Headings:** Use `<h1>`, `<h2>`, and `<h3>` for distinct document and clause titles.
2. **Text Blocks:** All standard text must be wrapped inside `<p>` paragraph tags.
3. **Lists:** Ordered lists (`<ol>`) and unordered lists (`<ul>`) should be used for numbered clauses, schedules, and definitions. Make sure to wrap items in `<li>`.
4. **Inline Formatting:**
   - Use `<strong>` or `<b>` for bold text (such as defined terms).
   - Use `<em>` or `<i>` for italics (such as Latin legal maxims).
   - Use `<u>` for underlined text if supported.

## Critical Output Rules for AI Agents

1. **Raw HTML Only:** The LLM's response output **MUST** be raw HTML. 
2. **NO Markdown Blocks:** Do **NOT** wrap the HTML inside Markdown code fences. 
   - ❌ **Bad:** ```html\n<p>Here is your drafted clause...</p>\n```
   - ✅ **Good:** `<p>Here is your drafted clause...</p>`
3. **Inline CSS Rules:** The editor strips most standard inline CSS attributes.
   - ❌ **Do NOT use** generic styles like `style="color: red; font-size: 20px;"`, or Tailwind classes.
   - ✅ **Exception for Alignment:** You **MAY** use `style="text-align: center|right|justify"` on `<p>`, `<h1>`, `<h2>`, and `<h3>` elements only. The editor natively parses this alignment.
4. **Hierarchy:** Ensure proper parent-child hierarchy! Do not nest block elements illegally (i.e. do not put `<p>` tags directly inside a `<ul>` without an `<li>`).

## Example Expected Output

```html
<h2>Confidentiality Clause</h2>
<p>The Receiving Party shall keep all <strong>Confidential Information</strong> strictly confidential and shall not, without the Disclosing Party's prior written consent, disclose such information to any third party.</p>
<ol>
  <li>This obligation shall survive the termination of this Agreement for a period of five (5) years.</li>
  <li>Confidential Information shall not include information that is publicly known at the time of disclosure.</li>
</ol>
```
